Документация для файла 1.php: (будет переименован)
 
Описание: 
Файл 1.php является обработчиком POST-запросов, принимает данные в формате JSON, обрабатывает их и возвращает данные в формате JSON. 
 
Использование: 
1. Отправьте POST-запрос на файл 1.php. 
2. В теле запроса передайте данные в формате JSON, содержащие ключ 'id'. 
3. В зависимости от значения ключа 'id', будет сформирован массив данных $data с информацией о продуктах. 
4. Массив $data будет преобразован в формат JSON и возвращен в ответ на запрос. 
 
Пример использования:
php
// Пример отправки POST-запроса на файл 1.php
// Передаем данные в формате JSON с ключом 'id' равным 1
$data = json_encode(['id' => 1]);

$ch = curl_init('1.php');
curl_setopt($ch, CURLOPT_POST, 1);
curl_setopt($ch, CURLOPT_POSTFIELDS, $data);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
curl_close($ch);

// Пример обработки ответа
$result = json_decode($response, true);
foreach ($result as $product) {
    echo "Имя: " . $product['name'] . "<br>";
    echo "Цена: " . $product['price'] . "<br>";
}
Примечания: 
- В данном примере файл 1.php возвращает предопределенные данные в зависимости от значения ключа 'id'. В реальном приложении, код может быть изменен для получения данных из базы данных или других источников. 
- Закомментированный код, который содержит запрос к базе данных, может быть использован для получения реальных данных о продуктах. В текущем состоянии, он не работает и нуждается в настройке под конкретную базу данных.

Документация для файла core.php

Описание: 
Файл содержит набор функций и настроек, которые используются в проекте. 
Включает в себя подключение к базе данных, функции для работы с сессиями, 
функции для перенаправления пользователя, функции для обработки ошибок, 
функции для выполнения запросов к базе данных и функции для проверки прав доступа пользователя. 
 
Функции: 
 
1.  session_start() : Функция для начала сессии. 
 
2.  new PDO() : Создание нового объекта PDO для подключения к базе данных MySQL. 
 
3.  loadComp($file) : Функция для загрузки компонентов. Принимает имя файла в качестве аргумента и подключает его, если файл существует. 
 
4.  redirect($url = false) : Функция для перенаправления пользователя на указанный URL или на предыдущую страницу (если URL не указан). Используется функция  header()  для отправки заголовка Location. 
 
5.  emptyValues($keyValues, $values) : Функция для проверки пустых значений. Принимает два аргумента: массив ключей  $keyValues  и массив значений  $values . Проверяет каждое значение в массиве  $values  и, если значение пусто, добавляет ошибку в массив  $_SESSION['errors'] . Возвращает массив  $arrValues  с заполненными значениями. 
 
6.  error($error, $string = 'errorForm') : Функция для обработки ошибок. Принимает сообщение об ошибке  $error  и опциональный аргумент  $string , который указывает, в какой массив  $_SESSION  сохранить ошибку. После сохранения ошибки, вызывает функцию  redirect() . 
 
7.  insert($table, $arrVal, $values) : Функция для выполнения запроса на вставку данных в таблицу базы данных. Принимает имя таблицы  $table , массив ключей  $arrVal  и массив значений  $values . Генерирует строку SET для запроса INSERT и выполняет запрос с использованием подготовленного выражения. 
 
8.  isUser($bool = false) : Функция для проверки, является ли пользователь зарегистрированным. Принимает опциональный аргумент  $bool , который указывает, нужно ли только проверить, является ли пользователь зарегистрированным или вернуть результат проверки. Если  $bool  установлен в  true , функция возвращает  true , если пользователь зарегистрирован, и  false  в противном случае. Если  $bool  не указан, функция перенаправляет пользователя на страницу 'index', если пользователь не зарегистрирован. 
 
9.  isAdmin($bool = false) : Функция для проверки, является ли пользователь администратором. Принимает опциональный аргумент  $bool , который указывает, нужно ли только проверить, является ли пользователь администратором или вернуть результат проверки. Если  $bool  установлен в  true , функция возвращает  true , если пользователь является администратором, и  false  в противном случае. Если  $bool  не указан, функция перенаправляет пользователя на страницу 'index', если пользователь не является администратором. 
 
Примечания: 
- В коде используется глобальная переменная  $config , которая не определена в данном фрагменте кода. Предполагается, что она определена в другом месте проекта. 
- Данный код предоставляет основные функции для работы с сессиями, базой данных и проверкой прав доступа. В реальном проекте может быть расширен и модифицирован в соответствии с требованиями.
